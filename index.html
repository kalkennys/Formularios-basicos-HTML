Aprenda as tags e técnicas mais usadas por desenvolvedores

O que é um Formulário?

Um formulário HTML é um elemento que permite aos usuários inserir dados e enviá-los para um servidor. Formulários são fundamentais para qualquer interação entre usuário e aplicação web, como login, cadastro, pesquisa e envio de informações.

💡 Dica: Formulários bem estruturados melhoram a experiência do usuário e facilitam a validação de dados do lado do cliente.

Tag Fundamental: <form>

<form></form>
Container que envolve todos os campos do formulário. Define como e onde os dados serão enviados.
<form action="processar.php" method="POST"> <!-- campos aqui --> </form>
Atributos principais:
  • action: URL onde o formulário será enviado
  • method: GET (visível na URL) ou POST (enviado discretamente)
  • name: Identificador do formulário
  • enctype: Tipo de codificação (importante para upload de arquivos)

Tag INPUT - A Mais Versátil

<input>
A tag mais importante! Cria diversos tipos de campos de entrada. Seu tipo é definido pelo atributo "type".
<!-- Exemplos de tipos --> <input type="text" placeholder="Digite seu nome"> <input type="email" placeholder="seu@email.com"> <input type="password" placeholder="Sua senha"> <input type="number" min="0" max="100"> <input type="date"> <input type="checkbox"> Concordo com os termos <input type="radio" name="opcao"> Opção 1 <input type="file" accept="image/*"> <input type="submit" value="Enviar"> <input type="reset" value="Limpar">
Tipos mais comuns:
  • text: Texto simples
  • email: Valida email automaticamente
  • password: Oculta caracteres
  • number: Apenas números
  • date: Seletor de data
  • checkbox: Múltiplas seleções
  • radio: Uma seleção entre várias
  • file: Upload de arquivos
  • submit: Botão para enviar

Tag LABEL - Acessibilidade Essencial

<label></label>
Associa um texto descritivo a um campo de entrada, melhorando acessibilidade e área clicável.
<label for="email">Email:</label> <input type="email" id="email" name="email">
Atributo: for - deve corresponder ao id do input

Tag SELECT - Lista de Opções

<select></select>
Cria um menu suspenso com opções pré-definidas. Ótimo para economizar espaço.
<select name="pais" required> <option value="">Selecione um país</option> <option value="br">Brasil</option> <option value="us">Estados Unidos</option> <option value="pt">Portugal</option> </select>
Atributos:
  • multiple: Permite múltiplas seleções
  • size: Número de opções visíveis

Tag TEXTAREA - Texto Longo

<textarea></textarea>
Para entradas de texto com múltiplas linhas. Ideal para comentários e mensagens.
<textarea name="mensagem" rows="5" cols="50" placeholder="Digite sua mensagem aqui..."> </textarea>
Atributos:
  • rows: Número de linhas visíveis
  • cols: Número de colunas
  • placeholder: Texto de exemplo

Tag BUTTON - Botões Interativos

<button></button>
Mais flexível que input type="submit". Permite conteúdo HTML dentro.
<button type="submit">Enviar Formulário</button> <button type="reset">Limpar Dados</button> <button type="button" onclick="alerta()">Clique-me</button>
Tipos:
  • submit: Envia o formulário
  • reset: Limpa todos os campos
  • button: Ação customizada com JavaScript

Outras Tags Importantes

<fieldset></fieldset>
Agrupa campos relacionados com uma borda visual. Melhora a organização.
<fieldset> <legend>Dados Pessoais</legend> <input type="text" placeholder="Nome"> <input type="email" placeholder="Email"> </fieldset>
<datalist>
Oferece sugestões enquanto o usuário digita (autocomplete).
<input list="cidades"> <datalist id="cidades"> <option value="São Paulo"> <option value="Rio de Janeiro"> <option value="Salvador"> </datalist>
<optgroup>
Agrupa opções dentro de um select em categorias.
<select> <optgroup label="Frutas"> <option>Maçã</option> <option>Banana</option> </optgroup> <optgroup label="Vegetais"> <option>Cenoura</option> <option>Alface</option> </optgroup> </select>

Atributos Universais Importantes

Atributo Descrição Exemplo
required Campo obrigatório <input type="email" required>
placeholder Texto de exemplo no campo <input placeholder="Digite algo">
readonly Campo não editável <input value="Fixo" readonly>
disabled Campo desativado (não envia dados) <input disabled>
name Identificador do campo (essencial!) <input name="email">
id Identificador único para CSS/JavaScript <input id="email-principal">
value Valor padrão do campo <input value="padrão">
pattern Validação por expressão regular <input pattern="[0-9]{3}-[0-9]{4}">

🌟 Boas Práticas para Formulários

1️⃣ Sempre use <label>

Melhora acessibilidade e aumenta a área clicável do campo.

2️⃣ Use atributo "name" em todos os campos

É essencial para que os dados sejam enviados corretamente no servidor.

3️⃣ Valide no servidor, mesmo validando no cliente

A validação HTML é apenas para UX. Sempre valide no backend!

4️⃣ Use tipos de input específicos

type="email", type="date", etc. proporcionam melhor experiência em dispositivos móveis.

5️⃣ Agrupe campos relacionados com <fieldset>

Melhora a organização visual e semântica do formulário.

6️⃣ Marque campos obrigatórios

Use required e indique visualmente com um asterisco (*).

7️⃣ Use placeholder com moderação

Placeholder não substitui label. Use ambos para melhor usabilidade.
