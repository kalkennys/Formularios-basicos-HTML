# 🤝 Como Contribuir

Obrigado por considerar contribuir para o **Guia Completo de Formulários HTML**! 

Contribuições de todos são bem-vindas e muito apreciadas! 🎉

---

## 📋 Tipos de Contribuição

### 🐛 Encontrou um Bug?

1. Verifique se já não foi reportado em [Issues](https://github.com/kalkennys/Formularios-basicos-HTML/issues)
2. Abra uma nova Issue descrevendo:
   - O que esperava acontecer
   - O que realmente aconteceu
   - Como reproduzir o problema
   - Navegador/Sistema operacional

### 📖 Quer Melhorar a Documentação?

- Corrija erros de digitação ou gramática
- Adicione exemplos que faltam
- Melhore a clareza das explicações
- Traduza para outro idioma

### ✨ Quer Adicionar uma Funcionalidade?

1. Abra uma Issue primeiro para discutir
2. Descreva por que seria útil
3. Espere feedback antes de começar a codificar

### 🎨 Quer Melhorar o Design?

- Sugestões de UI/UX são bem-vindas
- Melhore a responsividade
- Adicione temas (claro/escuro)

## 🚀 Processo de Contribuição

### 1. Fork o Projeto

```bash
# Clique no botão "Fork" no GitHub
# Depois clone seu fork:
git clone https://github.com/SEU_USUARIO/Formularios-basicos-HTML.git
cd Formularios-basicos-HTML
```

### 2. Crie uma Branch

```bash
git checkout -b feature/sua-funcionalidade
# ou
git checkout -b fix/seu-bug
```

**Nomes recomendados:**
- `feature/adicionar-validacao-js`
- `fix/corrigir-responsive-mobile`
- `docs/melhorar-documentacao`
- `style/refatorar-css`

### 3. Faça suas Mudanças

- Mantenha o código limpo e bem comentado
- Siga o padrão do projeto
- Teste no navegador

### 4. Commit suas Mudanças

```bash
git add .
git commit -m "Descrição clara da mudança"
```

**Bons exemplos de mensagens:**
- `feat: adicionar validação de email com regex`
- `fix: corrigir responsividade em dispositivos móveis`
- `docs: melhorar exemplos de uso`
- `style: refatorar CSS com melhor organização`

### 5. Push para sua Branch

```bash
git push origin feature/sua-funcionalidade
```

### 6. Abra um Pull Request

1. Vá para o repositório original
2. Clique em "Pull Requests"
3. Clique em "New Pull Request"
4. Selecione sua branch
5. Descreva o que você mudou e por quê
6. Aguarde review!

---

## ✅ Checklist do Pull Request

Antes de enviar, verifique:

- [ ] Testei no navegador (Chrome, Firefox, Safari)
- [ ] O código segue o estilo do projeto
- [ ] Adicionei comentários onde necessário
- [ ] A documentação foi atualizada
- [ ] Sem console errors ou warnings
- [ ] Responsivo em mobile (testar no DevTools)
- [ ] Acessibilidade não foi comprometida

## 📐 Guia de Estilo

### HTML

```html
<!-- Use nomes semânticos -->
<section class="legivel">
  <h2>Título Claro</h2>
  <!-- Indente corretamente -->
  <div>
    <!-- 2 espaços -->
  </div>
</section>
```

### CSS

```css
/* Use classes descritivas */
.formulario-secao {
  /* Properties em ordem lógica */
  display: flex;
  gap: 1rem;
  /* Use variáveis quando possível */
  background-color: var(--cor-primaria);
}
```

### JavaScript

```javascript
// Use nomes claros
function enviarFormulario(evento) {
  evento.preventDefault();
  // Comente lógica complexa
}

// Use const por padrão
const dados = new FormData(formulario);
```
